# Excel 自动报表助手 Demo 产品文档

## 1. 产品定位

**产品名称**：Excel 自动报表助手

**一句话说明**：上传一份订单、销售或经营明细 Excel，系统自动清洗数据、识别异常、生成汇总表和可直接发给老板的经营周报。

**目标形态**：先做一个可演示、可交付的小型工具，不做大平台。首版重点解决一个高频场景：中小团队每周手工整理 Excel、做销售统计、写周报的问题。

## 2. 目标客户

### 2.1 第一批客户

- 电商店主、抖店/淘宝/拼多多运营
- 小型贸易公司老板
- 本地门店老板
- 销售团队负责人
- 行政、财务、运营人员

### 2.2 典型用户画像

**用户 A：小公司老板**

- 每周需要看销售额、订单数、退款、利润、客户来源等数据。
- 员工发来的 Excel 格式不稳定，需要反复追问。
- 不想自己写公式，也不想额外招数据人员。

**用户 B：运营/财务人员**

- 每天或每周要合并多个 Excel。
- 常见工作包括删除空行、统一日期格式、统计金额、筛选异常订单、做透视表、写汇报。
- 最大痛点是重复、耗时、容易出错。

## 3. 核心痛点

1. **整理数据耗时**
   - 手工清洗、筛选、复制粘贴、做透视表，通常需要 30 分钟到 3 小时。

2. **公式和格式容易出错**
   - 日期、金额、店铺、渠道、商品名格式不统一，导致统计结果不准。

3. **老板看不懂原始表**
   - 老板要的是结论，例如本周卖了多少、哪个产品最好、哪里异常，而不是几千行明细。

4. **每周都重复**
   - 不是一次性需求，适合做成标准服务包或按月维护。

## 4. 产品目标

### 4.1 Demo 目标

让客户在 3 分钟内看懂价值：

> 原来需要人工整理 1 小时的 Excel，现在上传文件后 30 秒生成汇总表、异常提醒和老板版周报。

### 4.2 商业目标

- 支持作为 999-4999 元定制服务的销售 demo。
- 可根据客户 Excel 模板快速改造成专属版本。
- 后续可升级为按月维护服务，例如每月 500-3000 元。

## 5. 使用场景

### 场景 1：销售周报自动生成

用户上传本周订单明细，系统自动输出：

- 总销售额
- 订单数量
- 客单价
- 退款金额
- 净销售额
- Top 10 商品
- Top 10 客户
- 各渠道销售占比
- 异常订单列表
- 一段可直接发给老板的周报文字

### 场景 2：多表合并

用户上传多个 Excel，例如不同店铺、不同员工、不同日期的数据，系统自动合并成统一格式，并生成汇总结果。

### 场景 3：异常数据检查

系统自动标记：

- 金额为空或为 0 的订单
- 退款金额大于销售金额的订单
- 缺少客户名、渠道、商品名的行
- 日期格式异常的行
- 重复订单号

## 6. MVP 功能范围

### 6.1 必做功能

1. **上传 Excel 文件**
   - 支持 `.xlsx`。
   - 首版只支持单文件上传。

2. **字段识别**
   - 支持识别以下字段：
     - 订单号
     - 下单日期
     - 客户名称
     - 商品名称
     - 销售渠道
     - 销售金额
     - 退款金额
     - 成本
     - 销售员/负责人

3. **数据清洗**
   - 删除完全空白行。
   - 统一日期格式。
   - 金额字段转为数字。
   - 自动补充缺失退款金额为 0。
   - 标记关键字段缺失的行。

4. **自动汇总**
   - 销售额汇总。
   - 订单数汇总。
   - 退款汇总。
   - 净销售额汇总。
   - 毛利估算：销售金额 - 退款金额 - 成本。
   - 按商品、渠道、负责人统计。

5. **异常检测**
   - 输出异常订单表。
   - 给出异常原因。

6. **生成老板版周报**
   - 用自然语言输出一段总结。
   - 包含本周表现、增长点、风险点和建议动作。

7. **导出结果 Excel**
   - 输出一个新的 Excel 文件。
   - 包含以下工作表：
     - `清洗后明细`
     - `销售汇总`
     - `商品排行`
     - `渠道分析`
     - `异常订单`
     - `老板周报`

### 6.2 暂不做功能

- 不做账号系统。
- 不做在线支付。
- 不做多人协作。
- 不做复杂 BI 看板。
- 不做所有行业通用字段适配。
- 不做完全自动理解任意 Excel 格式。

首版重点是演示“上传 Excel -> 自动生成经营报表”的闭环。

## 7. 输入数据要求

Demo 使用一份标准订单 Excel，建议字段如下：

| 字段名 | 示例 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 订单号 | SO20260612001 | 是 | 用于识别重复订单 |
| 下单日期 | 2026-06-12 | 是 | 用于按时间统计 |
| 客户名称 | 张三 | 否 | 用于客户排行 |

| 销售渠道 | 抖音 | 是 | 用于渠道统计 |
| 销售金额 | 199 | 是 | 核心统计字段 |
| 退款金额 | 0 | 否 | 为空时按 0 处理 |

| 负责人 | 李四 | 否 | 用于团队统计 |

## 8. 输出结果

### 8.1 销售汇总

输出指标：

- 总订单数
- 有效订单数
- 总销售额
- 总退款金额
- 净销售额
- 总成本
- 估算毛利
- 客单价
- 退款率

### 8.2 商品排行

输出字段：

- 商品名称
- 销售额
- 订单数
- 退款金额
- 净销售额
- 估算毛利
- 销售占比

### 8.3 渠道分析

输出字段：

- 渠道名称
- 销售额
- 订单数
- 客单价
- 退款率
- 销售占比

### 8.4 异常订单

输出字段：

- 行号
- 订单号
- 异常类型
- 异常说明
- 建议处理方式

异常类型示例：

- 缺少订单号
- 缺少商品名称
- 销售金额为空
- 退款金额大于销售金额
- 重复订单号
- 日期格式异常

### 8.5 老板周报

示例输出：

> 本周共统计 1286 笔订单，总销售额 236,800 元，退款金额 12,300 元，净销售额 224,500 元，估算毛利 86,200 元。销售额最高的商品是“防晒喷雾”，贡献 28.4% 的销售额。抖音渠道表现最好，销售额占比 46.7%。本周发现 17 条异常订单，主要问题是重复订单号和退款金额异常，建议运营先核对异常订单后再进行财务结算。

## 9. 用户流程

1. 用户打开工具页面。
2. 点击上传 Excel。
3. 系统读取并预览前 10 行数据。
4. 系统识别字段并提示是否缺少必要字段。
5. 用户点击“生成报表”。
6. 系统生成汇总结果和老板周报。
7. 用户下载处理后的 Excel。

常见提示：

## 16. 风险和处理

### 16.1 Excel 格式不统一

处理方式：

- Demo 先限制标准模板。
- 对真实客户按模板定制，不承诺任意 Excel 都能完美识别。

### 16.2 客户需求无限扩展

处理方式：

- 报价前明确支持的字段、报表和修改次数。
- 新增复杂规则单独报价。

### 16.3 数据准确性要求高

处理方式：

- 对关键指标展示计算公式。
- 输出异常明细，方便人工复核。
- 首版定位为辅助报表工具，不替代财务审计。

## 17. 推荐首版 demo 数据

建议准备一份 300-1000 行的模拟订单表，包含：

- 5 个销售渠道
- 20 个商品
- 100 个客户
- 3-5 名负责人
- 10-20 条异常数据
- 部分退款订单
- 部分缺失字段
- 部分重复订单号

这样演示时能明显展示自动清洗、汇总和异常检测的价值。
